## ☕ Java 8 Support

The biggest change in this release: **the SDK now supports Java 8 and above.**

Previously, the SDK required Java 21, which excluded a significant portion of the Minecraft server ecosystem, especially older 1.8.x servers that still run on legacy JDKs.

Starting from `1.0.17`, the SDK is fully compatible with:

| Java Version | Supported |
| :--- | :--- |
| Java 8 | ✅ |
| Java 11 | ✅ |
| Java 17 | ✅ |
| Java 21 | ✅ |

This change required a complete rework of how the fat JAR is assembled. More on that below:

## 🐛 Bug Fixes

This release resolves **three critical issues** that were introduced during the Java 8 migration. They appeared sequentially — each fix revealed the next — but all share the same root cause: how Kotlin libraries were being bundled and relocated inside the SDK JAR.

<details>
<summary><strong>Fix #1 — `ExceptionInInitializerError` on authentication (1.0.15 → 1.0.16)</strong></summary>

**What happened:**
The `kotlin-reflect` library was being relocated inside the fat JAR (e.g. `kotlin.*` → `dev.efelleto.diogenes.libs.kotlin.*`). This broke an internal mechanism in `kotlin-reflect` that loads built-in type metadata from `.kotlin_builtins` resource files using hardcoded classpath paths.

**Error:**
```
Caused by: java.lang.IllegalStateException: Resource not found in classpath:
dev/efelleto/diogenes/libs/kotlin/...kotlin_builtins
```

**Fix:**
`kotlin-reflect` is now fully excluded from the fat JAR. Since it modifies internal JVM behavior and relies on hardcoded resource paths, it cannot be safely relocated.

</details>

<details>
<summary><strong>Fix #2 — `ClassNotFoundException: kotlinx.coroutines.CoroutineScope` (1.0.16 → 1.0.17)</strong></summary>

**What happened:**
After excluding `kotlin-reflect`, `kotlinx-coroutines` was also missing from the JAR. The Ktor CIO engine depends heavily on coroutines, and since the target servers (1.8.8 vanilla) don't provide Kotlin at runtime, the class was simply not found.

**Error:**
```
java.lang.ClassNotFoundException: kotlinx.coroutines.CoroutineScope
```

**Fix:**
`kotlinx-coroutines-core` is now bundled inside the fat JAR without relocation, ensuring it's always available regardless of the server environment.

</details>

<details>
<summary><strong>Fix #3 — Banner log pollution on server startup (1.0.17)</strong></summary>

**What happened:**
The Diogenes ASCII banner was being printed inside a `runTaskLater` (10 tick delay), which caused Spigot's own startup messages to interleave with the banner output in the console.

**Before:**
```
__________________________________________________________________________________
    _____^_
   ...
__________________________________________________________________________________
[INFO]: The server will be available in 4 seconds.   ← intruder
[INFO]: The server will be available in 3 seconds.   ← intruder
[INFO]: AUTH: Successfully authenticated.
```

**After:**
```
__________________________________________________________________________________
    _____^_
   ...
__________________________________________________________________________________

[INFO]: The server will be available in 4 seconds.
[INFO]: The server will be available in 3 seconds.
[INFO]: AUTH: Successfully authenticated.
```

**Fix:**
The banner is now printed synchronously during `onEnable()`, before any async task is scheduled.

</details>

## 📦 SDK Build Rework

The entire fat JAR assembly strategy was redesigned. Here's the final configuration that makes everything work cleanly:

| Library | Strategy | Reason |
| :--- | :--- | :--- |
| `kotlin-stdlib` | Bundled, not relocated | Server doesn't provide it |
| `kotlin-reflect` | Excluded | Breaks with any relocation |
| `kotlinx-coroutines` | Bundled, not relocated | Required by Ktor CIO |
| `kotlinx.atomicfu` | Bundled + relocated | Transitive dep, may conflict |
| `io.ktor` | Bundled + relocated | Avoid classpath conflicts |
| `com.google.gson` | Bundled + relocated | Avoid classpath conflicts |

## 📚 Documentation Overhaul

All documentation pages were reviewed and updated to reflect the current state of the project:

- **Getting Started:** Java requirement corrected to **Java 8+**, Gradle snippet fixed to proper Kotlin DSL syntax.
- **Integration:** Fully rewritten with a parameter reference table, step-by-step authentication flow, and a complete error handling table.
- **Installation:** Fixed inconsistency between `config.yml` and `settings.yml` references. Configuration block is now a proper YAML example.
- **Discord Commands:** Fixed header hierarchy and standardized formatting.
- **Security Flow:** Minor formatting cleanup.
