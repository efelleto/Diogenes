1.0.20

March 13, 2026 · 5 min read
Eduardo Felleto
Head Developer

A lot happened since the last devlog. 1.0.20 ships the most important fix in the project's history, the Agora community server is now live, and the myPlugins storefront is officially open. Let's break it all down.

1.0.20 - The HWID Overhaul​

If you've been using Diogenes for a while, you may have noticed an annoying problem: valid licenses occasionally triggering a hardware mismatch after a reboot, a system update, or a simple user switch. This is known as a False Positive: The system blocking a legitimate user for no real reason.

Version 1.0.20 eliminates this entirely. The full change is in SecurityUtils.kt.

The Problem: Volatile Data​

The old HWID was built using data that changes without the user doing anything:

  • user.name: if the server was started as root and then as a regular user, the hash changed.
  • os.version: a kernel update or Windows patch was enough to break the license.
  • Network interface ordering: Java reads network adapters in the order the OS delivers them. On reboot, if Wi-Fi responded 1ms before Ethernet, the MAC address list inverted and the hash changed.

All of these caused real users to get locked out of plugins they legitimately purchased.

The Fix: Static Anchors​

🔧 Sorted MAC List

Previously, MAC addresses were appended directly as the OS returned them:

// before — order-dependent, breaks on reboot
hwData.append(mac.joinToString("") { "%02x".format(it) })

Now they are collected into a list, sorted alphabetically, then appended:

// after — stable regardless of boot order
macList.add(mac.joinToString("") { "%02x".format(it) })
macList.sort()
macList.forEach { hwData.append(it) }

Also added explicit filtering for VirtualBox and Docker interfaces, which change on every container restart:

if (netInterface.isLoopback || netInterface.isVirtual 
    || netInterface.name.contains("vboxnet") 
    || netInterface.name.contains("docker")) continue
🔧 Removal of volatile variables

user.name and os.version have been removed from the HWID calculation entirely. The system now keeps only stable, hardware-level properties:

// removed — volatile
hwData.append(System.getProperty("os.version"))
hwData.append(System.getProperty("user.name"))

// kept — stable
hwData.append(System.getProperty("os.name"))
hwData.append(System.getProperty("os.arch"))
hwData.append(Runtime.getRuntime().availableProcessors())
🔧 Linux machine-id

On Linux, the SDK now reads /etc/machine-id, a file generated at OS installation time that acts as a permanent unique identifier. It survives reboots, user switches, and system updates:

if (System.getProperty("os.name").lowercase().contains("linux")) {
    val machineId = File("/etc/machine-id")
    if (machineId.exists()) hwData.append(machineId.readText().trim())
}
🔧 macOS IOPlatformUUID

On macOS, the SDK now captures the IOPlatformUUID directly from the system's hardware registry. This identifies the logic board, making it virtually impossible to spoof without physically replacing hardware:

} else if (System.getProperty("os.name").lowercase().contains("mac")) {
    try {
        val process = Runtime.getRuntime().exec("ioreg -rd1 -c IOPlatformExpertDevice")
        val uuid = process.inputStream.bufferedReader().readText()
            .split("IOPlatformUUID")[1]
            .split("\"")[2]
        hwData.append(uuid)
    } catch (e: Exception) {}
}

The Result​

ScenarioBefore 1.0.20After 1.0.20
Server reboot⚠️ Possible mismatch✅ Stable
Kernel / OS update⚠️ Possible mismatch✅ Stable
Running as root vs user⚠️ Possible mismatch✅ Stable
Network adapter reorder⚠️ Possible mismatch✅ Stable
VirtualBox / Docker interfaces⚠️ Possible mismatch✅ Filtered out

The Diogenes SDK now has a hardware memory that is stable, fair, and robust. You can sell licenses with confidence knowing that your customers won't be locked out of software they paid for.

Discord server released​

The official Diogenes community server, Agora, is now open.

Agora is built around mutual help and knowledge sharing. Whether you're setting up your first Diogenes server, debugging an integration, or just want to follow the project's progress, this is the place.

What you'll find:

  • #announcements and #updates to stay on top of every release
  • #faq with answers to the most common questions
  • #community-help for support from the community
  • #contact via ModMail for private or commercial inquiries

Volunteer moderator applications will be opening soon. If you're active in the community and want to help keep Agora a great place, stay tuned to #announcements.

Supporter program is open​

Diogenes is and will always be free and open-source. No ads, no subscriptions, no hidden fees.

If the project has been useful to you, you can now support its development through Ko-fi or PIX. Every contribution, no matter the size, helps keep the ecosystem alive and growing.

Supporters receive:

  • Supporter role in Agora
  • Access to the exclusive #supporters chat
  • Access to #colors to set a custom name color via Jeb BOT
  • Discount on myPlugins products
  • Priority support in #contact
  • Your name in the project credits on GitHub

Introducing myPlugins​

myPlugins is the storefront for plugins built, tested, and protected with Diogenes.

Every plugin available through myPlugins is production-ready and comes with the same infrastructure standard you already know from the Diogenes ecosystem. Custom development is also available, open a thread in #contact to discuss your project.

Browse available products in the #products channel inside Agora.
